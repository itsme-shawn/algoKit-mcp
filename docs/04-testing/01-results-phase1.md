## 구현 완료 내역

### 1. API 타입 정의 (`src/api/types.ts`)

**목적**: solved.ac API 응답 타입 및 에러 클래스 정의

#### 주요 타입
```typescript
- Problem          // 백준 문제 정보
- Tag              // 알고리즘 태그
- SearchResult     // 검색 결과 (문제 또는 태그)
- SearchParams     // 검색 파라미터
```

#### 커스텀 에러 클래스 (6개)
| 에러 클래스 | HTTP 코드 | 용도 |
|------------|-----------|------|
| `SolvedAcAPIError` | - | 기본 에러 클래스 |
| `ProblemNotFoundError` | 404 | 문제를 찾을 수 없음 |
| `TimeoutError` | - | 요청 시간 초과 |
| `NetworkError` | - | 네트워크 에러 |
| `RateLimitError` | 429 | Rate limit 초과 |
| `InvalidInputError` | - | 입력 검증 실패 |

#### 타입 가드
- `isProblem(obj)`: Problem 타입 검증
- `isTag(obj)`: Tag 타입 검증

**테스트 커버리지**: 100% (32개 테스트)

### 2. API 클라이언트 (`src/api/solvedac-client.ts`)

**목적**: solved.ac API와 통신하는 HTTP 클라이언트

#### 핵심 기능
- **자동 재시도 로직**: 최대 3회, 지수 백오프 (1초 → 2초 → 3초)
- **내장 캐싱**: 문제 상세 (1시간 TTL), 검색 결과 (10분 TTL)
- **타임아웃 처리**: 10초 제한
- **포괄적인 에러 처리**: HTTP 상태 코드별 커스텀 에러

#### 공개 메서드
1. **`searchProblems(params)`**: 문제 검색
   - 레벨 범위 검증 (1-30)
   - 페이지 번호 검증
   - 쿼리 파라미터 생성 및 URL 인코딩

2. **`getProblem(problemId)`**: 문제 상세 조회
   - problemId 양수 검증
   - 404 시 ProblemNotFoundError

3. **`searchTags(query)`**: 태그 검색
   - 빈 문자열 검증
   - items 배열 반환

4. **`clearCache()`**: 캐시 수동 초기화

**테스트 커버리지**: 95.5% (34개 테스트)

### 4. 캐시 시스템 (`src/utils/cache.ts`)

**목적**: 인메모리 TTL 기반 캐시 구현

#### Cache<T> 클래스
- **제네릭 타입 지원**: 타입 안전한 캐시
- **TTL 기반 만료**: 자동 만료 처리
- **LRU 정책**: 최대 크기 제한 (선택사항)

#### 주요 메서드
```typescript
- set(key, value, ttl?)    // 값 저장
- get(key)                 // 값 조회 (만료 시 undefined)
- has(key)                 // 존재 여부 확인
- delete(key)              // 삭제
- clear()                  // 전체 삭제
- cleanup()                // 만료 항목 제거
- size()                   // 캐시 크기
- getStats()               // 통계 (hits, misses, size)
```

#### 싱글톤 인스턴스
- `problemCache`: 문제 상세 정보 (TTL: 1시간)
- `searchCache`: 검색 결과 (TTL: 10분)

## 주요 설계 결정사항

### 1. 간소화된 타입 구조
초기 복잡한 타입 정의를 간소화하여 핵심 타입만 유지했습니다. 이로 인해 코드 가독성과 유지보수성이 향상되었습니다.

### 2. 클라이언트 내장 캐싱
`SolvedAcClient` 클래스 내부에 캐싱 로직을 통합하여, API 호출 시 자동으로 캐시 확인 및 저장이 이루어지도록 했습니다. 이로 인해 중복 API 호출을 방지하고 성능을 개선했습니다.

### 3. 지수 백오프 재시도
네트워크 에러 발생 시 최대 3회까지 재시도하며, 재시도 간격은 1초, 2초, 3초로 지수적으로 증가합니다. 이는 일시적인 네트워크 문제를 자동으로 해결합니다.

### 4. 포괄적인 에러 처리
6가지 커스텀 에러 클래스를 정의하여 에러 유형을 명확히 구분하고, 각 에러에 적절한 처리를 할 수 있도록 했습니다.

### 5. 입력 검증
API 클라이언트 메서드에서 입력값을 사전에 검증하여 잘못된 요청이 API로 전송되는 것을 방지했습니다.
