# BOJ 학습 도우미 MCP Server

**버전**: 2.0 (Keyless Architecture)
**작성일**: 2026-02-13
**마지막 업데이트**: 2026-02-13
**상태**: Phase 3 완료 (Keyless 아키텍처 구현)

---

## 1. 프로젝트 개요

### 1.1 제품 비전

BOJ 학습 도우미는 **백준 온라인 저지(Baekjoon Online Judge) 알고리즘 문제를 효과적으로 학습**할 수 있도록 돕는 AI 기반 MCP(Model Context Protocol) 서버입니다.

사용자가 Claude와 자연스럽게 대화하며 문제를 검색하고, 적절한 난이도의 힌트를 받고, 해결 후 체계적인 복습 문서를 생성할 수 있는 통합 학습 환경을 제공합니다.

### 1.2 한 줄 정의

**백준 ID 기반으로 풀이 이력을 분석하고, 학습 리포트와 문제별 복기 피드백 파일을 자동 생성하는 개인 알고리즘 학습 분석 서버**

### 1.3 문제 정의

**현재 BOJ 학습의 문제점**:

1. **전략 없는 문제 풀이**
   - 체계적인 학습 계획 없이 무작위로 문제 선택
   - 현재 수준에 맞지 않는 문제로 시간 낭비

2. **취약 알고리즘 인지 불가**
   - 어떤 알고리즘이 약한지 정량적으로 파악하기 어려움
   - 같은 유형의 문제를 반복적으로 틀림

3. **티어 정체 구간 돌파 전략 부재**
   - Silver/Gold 구간에서 장기간 정체
   - 다음 단계로 올라가기 위한 명확한 학습 방향성 부족

4. **복기 기록 자동화 없음**
   - 해결한 문제를 다시 보지 않음
   - 풀이 과정에서 배운 점을 기록하지 않음

5. **문제별 개선점 정리 안 함**
   - 왜 틀렸는지, 어떻게 개선할 수 있는지 분석 부재
   - 비슷한 실수를 반복

**결과적으로**:
- 많이 풀어도 실력이 체계적으로 상승하지 않음
- Gold 진입에서 정체
- 학습 효율성 저하

**타겟 사용자**:
- 알고리즘 문제 해결 능력을 향상시키고자 하는 학생 및 취업 준비생
- 코딩 테스트를 준비하는 개발자
- BOJ를 통해 알고리즘을 체계적으로 학습하고자 하는 모든 사용자
- 특히 **티어 정체 구간에서 돌파구를 찾는 학습자**

### 1.4 제품 목표

**🎯 Primary Goal (필수 기능)**:
1. **백준 ID 기반 전체 학습 분석 리포트 자동 생성**
   - 풀이 이력 종합 분석
   - 취약 알고리즘 정량화
   - 티어 정체 구간 탐지
   - 맞춤 학습 전략 제안

2. **특정 문제 풀이 후 복기 파일 자동 생성**
   - 문제별 핵심 개념 정리
   - 상세 풀이 비교
   - 개선 포인트 제시
   - 배운 점 정리 섹션

**🎯 Secondary Goal (부가 기능)**:
- 맞춤 문제 추천 시스템
- 단계별 힌트 제공
- 학습 진척도 시각화
- 알고리즘별 취약도 점수

**비목표 (Out of Scope for MVP)**:
- 실시간 코드 실행 및 채점
- 사용자 계정 및 로그인 시스템
- 문제 풀이 영상이나 강의 콘텐츠
- 소셜 기능 (친구, 그룹 스터디 등)

## 2. 핵심 기능 명세

### 2.1 문제 검색 및 발견 (Problem Search & Discovery)

#### 기능 1: `search_problems` - 문제 검색
**설명**: 다양한 필터를 사용하여 BOJ 문제를 검색합니다.

**기능 요구사항**:
- 키워드로 문제 검색 (제목, 번호, 태그)
- 난이도 범위 필터링 (Bronze ~ Ruby, 레벨 1-30)
- 알고리즘 태그 필터링 (DP, 그리디, 그래프 등)
- 정렬 옵션 (난이도, 문제 번호, 평균 시도 횟수)
- 페이지네이션 지원

**입력**:
```typescript
{
  query?: string;           // "이분 탐색", "1000", "투 포인터" 등
  level_min?: number;       // 1-30 (예: 11 = Gold V)
  level_max?: number;       // 1-30 (예: 15 = Gold I)
  tag?: string;             // "dp", "greedy", "graphs" 등
  sort?: "level" | "id" | "average_try";
  direction?: "asc" | "desc";
  page?: number;
}
```

**출력**:
```typescript
{
  count: number;            // 총 결과 개수
  problems: [
    {
      problemId: number;
      title: string;        // 한글 제목
      level: number;        // 1-30
      tierName: string;     // "Gold III"
      tags: string[];       // ["dp", "greedy"]
      acceptedCount: number;
      averageTries: number;
    }
  ]
}
```

**사용 시나리오**:
- "Gold 티어의 DP 문제를 찾고 싶어"
- "이분 탐색 문제 중에서 쉬운 것부터 보여줘"
- "평균 시도 횟수가 적은 Silver 문제 추천해줘"

#### 기능 2: `get_problem` - 문제 상세 정보
**설명**: 특정 문제의 상세 정보를 조회합니다.

**기능 요구사항**:
- 문제 ID로 메타데이터 조회
- BOJ 문제 링크 제공
- 통계 정보 (해결한 사용자 수, 평균 시도)
- 알고리즘 태그 목록

**입력**:
```typescript
{
  problem_id: number;  // 예: 1000
}
```

**출력**:
```typescript
{
  problemId: number;
  titleKo: string;
  titleEn: string;
  level: number;
  tierName: string;
  tags: Array<{
    key: string;
    displayName: string;
  }>;
  acceptedUserCount: number;
  averageTries: number;
  bojLink: string;  // https://www.acmicpc.net/problem/1000
}
```

**사용 시나리오**:
- "1927번 문제에 대해 알려줘"
- "이 문제의 난이도와 태그가 뭐야?"

#### 기능 3: `search_tags` - 알고리즘 태그 검색
**설명**: 알고리즘 태그를 키워드로 검색합니다.

**기능 요구사항**:
- 태그 키워드 검색
- 다국어 표시명 제공
- 각 태그의 문제 수 표시

**입력**:
```typescript
{
  query: string;  // "dynamic", "그래프" 등
}
```

**출력**:
```typescript
{
  tags: Array<{
    key: string;              // "dp"
    displayNameKo: string;    // "다이나믹 프로그래밍"
    displayNameEn: string;    // "Dynamic Programming"
    problemCount: number;     // 2847
  }>
}
```

**사용 시나리오**:
- "다이나믹 프로그래밍 관련 태그 검색해줘"
- "그래프 알고리즘에는 어떤 태그들이 있어?"

### 2.2 학습 지원 (Learning Support)

#### 기능 4: `analyze_problem` - 문제 분석 및 구조화된 힌트 데이터 제공 (Keyless)
**설명**: 문제를 분석하여 구조화된 힌트 데이터를 JSON으로 제공합니다. Claude Code가 이를 자연어로 변환합니다.

**Keyless 아키텍처**:
- **MCP 서버**: 결정적 JSON 데이터만 반환
- **Claude Code**: 자연어 생성 담당
- **Zero Configuration**: API 키 불필요

**기능 요구사항**:
- 3단계 힌트 포인트 시스템
  - **레벨 1 (Pattern)**: 알고리즘 문제 분석 (key + detail)
  - **레벨 2 (Insight)**: 핵심 아이디어 (key + detail + example)
  - **레벨 3 (Strategy)**: 구현 전략 (key + steps[])
- 정적 힌트 패턴 매핑 (`HINT_PATTERNS`)
- 알고리즘별 사전 정의된 힌트 (dp, greedy, graphs, math 등)
- 난이도 컨텍스트 (티어, 백분위, 이모지)
- 제약사항 및 주의사항 자동 생성
- 유사 문제 추천 (±2 티어 범위)

**입력**:
```typescript
{
  problem_id: number;
  include_similar?: boolean;  // 유사 문제 추천 포함 (기본값: true)
}
```

**출력** (JSON):
```typescript
{
  problem: {
    problemId: number;
    titleKo: string;
    level: number;
    tags: Tag[];
    // ... 문제 메타데이터
  },
  difficulty: {
    tier: string;           // "Silver II"
    level: number;          // 9
    emoji: string;          // "🟡"
    percentile: string;     // "초급 (상위 70-80%)"
    context: string;        // "Silver 난이도의 DP 문제"
  },
  algorithm: {
    primary_tags: string[];              // ["다이나믹 프로그래밍", "이분 탐색"]
    tag_explanations: Record<string, string>;  // 태그별 설명
    typical_approaches: string[];         // ["Bottom-up DP", "Top-down DP"]
    time_complexity_typical: string;      // "O(N) ~ O(N²)"
    space_complexity_typical: string;     // "O(N) ~ O(N²)"
  },
  hint_points: [
    {
      level: 1,
      type: "pattern",
      key: "동적 프로그래밍",
      detail: "DP의 전형적인 최적 부분 구조를 가진 문제입니다..."
    },
    {
      level: 2,
      type: "insight",
      key: "상태 정의와 점화식",
      detail: "dp[i]의 의미를 명확히 정의하고...",
      example: "dp[i] = min(dp[i-1], dp[i/2], dp[i/3]) + 1"
    },
    {
      level: 3,
      type: "strategy",
      key: "Bottom-up 또는 Top-down 구현",
      steps: [
        "1. 상태 정의 (dp[i]의 의미)",
        "2. 초기값 설정 (base case)",
        "3. 점화식 구현",
        "4. 계산 순서 결정",
        "5. 최종 답 반환"
      ]
    }
  ],
  constraints: [...],
  gotchas: [...],
  similar_problems: [...]
}
```

**힌트 생성 전략**:

**레벨 1 예시** (문제: 최단 경로 찾기):
```
이 문제는 그래프 탐색 문제입니다.
두 지점 사이의 최단 경로를 찾아야 하는데,
가중치가 없는 그래프에서 사용할 수 있는
기본적인 탐색 알고리즘을 생각해보세요.
```

**레벨 2 예시**:
```
BFS(너비 우선 탐색)를 사용하면 좋습니다.
BFS는 시작점에서 가까운 노드부터 차례로 탐색하므로,
처음 도착한 경로가 자동으로 최단 경로가 됩니다.
큐 자료구조를 사용하여 방문할 노드를 관리하세요.
```

**레벨 3 예시**:
```
알고리즘 단계:
1. 시작 노드를 큐에 넣고 방문 표시
2. 큐에서 노드를 꺼내고, 인접한 미방문 노드들을 큐에 추가
3. 각 노드까지의 거리를 기록 (이전 노드 거리 + 1)
4. 목표 노드에 도달할 때까지 2-3 반복
5. 목표 노드의 기록된 거리가 최단 거리

시간 복잡도: O(V + E) (V: 노드 수, E: 간선 수)
```

**사용 시나리오**:
- "이 문제 어떻게 접근해야 할지 막막해. 힌트 줘"
- "DP라는 건 알겠는데 상태 정의를 모르겠어. 더 자세한 힌트 필요해"

#### 기능 5: `generate_review_template` - 복습 템플릿 및 가이드 제공 (Keyless)
**설명**: 복습 문서 템플릿과 작성 가이드 프롬프트를 제공합니다. Claude Code가 대화형으로 복습 문서를 작성합니다.

**Keyless 아키텍처**:
- **MCP 서버**: 템플릿 + 가이드 프롬프트 반환
- **Claude Code**: 사용자와 대화하며 복습 문서 작성
- **Zero Configuration**: API 키 불필요

**기능 요구사항**:
- 마크다운 템플릿 생성 (문제 정보 자동 포함)
- 빈 섹션 제공 (풀이 접근법, 복잡도, 인사이트, 어려웠던 점)
- 관련 문제 자동 추천 (±2 티어 범위, 같은 태그)
- 작성 가이드 프롬프트 생성 (Claude Code가 사용)
- 해결 날짜 자동 추가

**입력**:
```typescript
{
  problem_id: number;
  user_notes?: string;  // (선택) 미리 작성한 메모
}
```

**출력** (JSON):
```json
{

  "problem_summary": {
    "problemId": 1927,
    "titleKo": "최소 힙",
    "tier": "Silver II",
    "tags": ["자료 구조", "우선순위 큐"],

    { "problemId": 1655, "titleKo": "가운데를 말해요", "tier": "Gold II" }

  "guide_prompt": "이 템플릿을 사용하여 복습 문서를 작성합니다. 사용자에게 다음을 질문하세요:\n1. 풀이 접근법: 어떤 알고리즘과 자료구조를 사용했나요?\n2. 시간/공간 복잡도: 계산해보셨나요?\n3. 핵심 인사이트: 이 문제에서 배운 핵심 개념은 무엇인가요?\n4. 어려웠던 점: 어떤 부분이 어려웠고, 어떻게 해결했나요?\n\n사용자 답변을 받으면 template의 해당 섹션에 채워넣고 최종 마크다운 문서를 반환하세요."

4. 최종 완성된 마크다운 문서 반환

**사용 시나리오**:
- "방금 푼 1927번 문제 복습 문서 만들어줘"

### 2.3 학습 분석 (Learning Analytics) - 핵심 기능

#### 기능 6: `analyze_user` - 백준 ID 기반 전체 학습 리포트 생성

**설명**: 백준 사용자 ID를 기반으로 전체 풀이 이력을 분석하고, 취약 알고리즘, 티어 정체 구간, 맞춤 학습 전략을 포함한 종합 리포트를 자동 생성합니다.

**기능 요구사항**:
- solved.ac API를 통한 사용자 풀이 이력 수집
- 티어별 문제 분포 분석
- 알고리즘별 풀이 비율 및 취약도 계산
- 정체 구간 탐지 (같은 티어 75% 이상)
- 2주 맞춤 학습 전략 제안
- 추천 문제 리스트 자동 생성

**입력**:
```typescript
{
  boj_id: string;           // 백준 사용자 ID
  analysis_depth?: "basic" | "detailed";  // 기본값: "detailed"
  recent_count?: number;    // 최근 N개 문제 분석 (기본값: 50)
}
```

### 2.4 문제별 복기 강화 (Problem Review Enhancement)

`create_review` 기능을 확장하여 AI 기반 코드 피드백과 개선점을 자동 생성합니다.

**확장 기능**:
1. **코드 분석** (선택사항)
   - 사용자 코드 입력 시 자동 분석
   - 변수명, 로직, 자료구조 선택 피드백
   - 시간복잡도 자동 계산

2. **다른 풀이 비교**
   - 최적 풀이 접근법 제시
   - 여러 풀이 방식 비교

3. **복습 체크리스트 자동 생성**
   - 핵심 개념 이해도 체크
   - 비슷한 유형 연습 필요 여부

**강화된 출력 예시**:
```markdown
## ✨ 코드 피드백

### 개선 포인트
1. **변수명 명확성**
   - `arr` → `numbers` (더 명확한 의미)
   - `dp` → `minCost` (목적 반영)

2. **로직 단순화**
   ```python
   # 현재 코드
   for i in range(len(arr)):
       if arr[i] > target:
           continue

   # 개선 코드
   for num in arr:
       if num > target:
           continue
   ```

3. **자료구조 최적화**
   - 리스트 대신 deque 사용 권장
   - 탐색 시간 O(n) → O(1)

                      ↓
                  문제 메타데이터 조회

                      ↓
                  마크다운 템플릿 채우기
                      ↓
                  사용자 ← Claude (마크다운 문서)
   ```

## 플랫폼 판별 규칙

MCP 도구 호출 및 스킬 실행 시 플랫폼(BOJ / 프로그래머스)을 다음 규칙으로 판별합니다.

| 입력 패턴 | 판별 결과 |
|-----------|-----------|
| `acmicpc.net` URL | BOJ 확정 |
| `school.programmers.co.kr` URL | Programmers 확정 |
| "백준", "boj", BOJ 티어명(브론즈/실버/골드 등) 키워드 포함 | BOJ 확정 |
| "프로그래머스", "pg", "레벨" 키워드 포함 | Programmers 확정 |
| 숫자만 입력 + 대화 맥락에서 플랫폼 확인됨 | 맥락의 플랫폼으로 진행 |
| **숫자만 입력 + 맥락 없음** | **반드시 플랫폼 되묻기** |

### 핵심 원칙

- **임의 추정 금지**: 맥락 없이 숫자만으로 플랫폼을 추정하지 않습니다.
- **맥락 활용**: 대화 중 이미 플랫폼이 언급된 경우 되묻지 않고 활용합니다.
- **URL 우선**: URL 입력은 항상 해당 플랫폼으로 확정합니다.

## 6. 통합 스킬: 플랫폼 자동 선택 규칙

### 6.1 스킬 개요

**스킬명**: `algokit`
**목적**: 사용자가 BOJ와 프로그래머스를 구분하지 않고 자연스럽게 코테 문제를 검색하고 학습할 수 있도록 플랫폼 자동 선택 및 라우팅 제공

**지원 플랫폼**:
- **백준 온라인 저지 (BOJ)**: 문제 분석, 힌트 생성, 복습 템플릿, 검색
- **프로그래머스 (Programmers)**: 문제 검색, 상세 조회

### 6.2 플랫폼 자동 선택 규칙

#### 규칙 1: 명시적 플랫폼 지정 시 → 자동 처리

사용자가 플랫폼을 명시하면 해당 플랫폼 도구를 자동 호출합니다.

**예시**:
- "백준 1000번" → BOJ `get_problem` 또는 `analyze_problem`
- "프로그래머스 42576번" → Programmers `get_programmers_problem`
- "백준에서 DP 문제 찾아줘" → BOJ `search_problems`
- "프로그래머스 해시 문제 찾아줘" → Programmers `search_programmers_problems`

**구현**:
- Claude Code가 스킬 설명을 기반으로 자연어 입력에서 플랫폼을 추출
- 적절한 MCP 도구 호출

#### 규칙 2: URL 제공 시 → 자동 파싱

사용자가 URL을 제공하면 도메인을 파싱하여 플랫폼을 자동 식별합니다.

**예시**:
- `https://www.acmicpc.net/problem/1000` → BOJ 문제 번호 1000
- `https://school.programmers.co.kr/learn/courses/30/lessons/42576` → 프로그래머스 문제 ID 42576

**구현**:
- `src/utils/url-parser.ts` (프로그래머스)
- `src/api/boj-scraper.ts` (BOJ)
- 도메인 기반 플랫폼 식별 → 해당 도구 호출

#### 규칙 3: 번호만 제공 시 → 사용자 확인

플랫폼 없이 숫자만 제공하면 사용자에게 플랫폼을 확인합니다.

**예시**:
- 사용자: "1000번"
- 시스템: "어느 플랫폼의 문제인가요? 백준 또는 프로그래머스를 선택해주세요."

**최적화 (향후)**:
- BOJ 범위 (1~30,000)면 BOJ 우선 시도 → 404면 프로그래머스 확인
- 프로그래머스는 보통 5자리 이상 (예: 42576)

#### 규칙 4: 일반 추천 요청 시 → BOJ 기본 + 추가 제안 ✅

플랫폼 없이 일반적인 추천을 요청하면 BOJ를 기본으로 제공하고, 프로그래머스 옵션을 제안합니다.

**예시**:
- 사용자: "코테 문제 추천해줘"
- 시스템:
  1. BOJ `search_problems` 결과 제공
  2. "프로그래머스 문제도 보시겠어요?" 메시지 추가
  3. 사용자가 "네"라고 하면 프로그래머스도 검색

**이유**: BOJ가 메타데이터가 풍부하고 API 지원이 우수하여 기본 플랫폼으로 선택

#### 규칙 5: 문제명으로 검색 시 → 두 플랫폼 모두 검색

문제 제목으로 검색하면 BOJ와 프로그래머스를 모두 검색하여 통합 결과를 제공합니다.

**예시**:
- 사용자: "피보나치 문제 찾아줘"
- 시스템:
  1. BOJ `search_problems` (query: "피보나치")
  2. 프로그래머스 `search_programmers_problems` (query: "피보나치")
  3. 마크다운 테이블로 통합 결과 제공 (플랫폼 컬럼 포함)

**출력 형식**:
```markdown
| 플랫폼 | 문제 번호 | 제목 | 난이도 | 태그 |
|--------|----------|------|--------|------|
| 백준 | 1003 | 피보나치 함수 | Silver III | DP |
| 프로그래머스 | 12945 | 피보나치 수 | Level 2 | - |
```

### 6.3 MCP 도구 매핑

#### BOJ 도구 (8개)
- `search_problems` - 문제 검색
- `get_problem` - 문제 상세 조회

#### 프로그래머스 도구 (2개)
- `search_programmers_problems` - 문제 검색 (Puppeteer, 3-5초)
- `get_programmers_problem` - 문제 상세 조회 (cheerio, 1-2초)

### 6.4 구현 영향도

#### 스킬 정의
- **위치**: `.claude/skills/algokit.md`
- **역할**: Claude Code가 자연어 입력을 분석하여 적절한 도구 선택
